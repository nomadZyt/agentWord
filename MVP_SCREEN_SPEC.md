# agentWord MVP Screen Specification

Last updated: 2026-05-29

## 1. 首屏目标

MVP 首屏必须优先传达“这是一个正在运转的 AI agent 园区”，而不是传统监控后台。

用户打开应用后，第一眼应该看到：

- 主场景中有房间、路径、agent、任务状态气泡和阻塞警报区。
- 右侧能扫描到所有任务列和关键任务详情。
- 底部或右下能看到事件流，知道 agent 刚刚发生了什么。
- 顶部能看到当前是模拟模式、基础监控模式，及本机检测到的 Codex / Claude 会话数量。

## 2. 首屏信息架构

桌面端 MVP 采用三层布局：

- 顶部状态栏：全局状态、真实会话数量、模拟 agent 数量、监控模式、隐私模式。
- 主内容区：左侧大面积 PixiJS 园区场景，右侧任务看板和详情面板。
- 底部事件流：展示最近事件，默认贴在场景底部，可折叠。

推荐桌面比例：

- 顶部状态栏：高度 48-56px。
- 主场景：占屏幕宽度约 65%-72%。
- 右侧面板：宽度 360-440px。
- 事件流：高度 120-180px，覆盖在场景底部或作为底部抽屉。

低保真结构：

```text
+--------------------------------------------------------------------------------+
| Global Status: Mode | Real Sessions | Sim Agents | Privacy | Time              |
+---------------------------------------------------------+----------------------+
|                                                         | Task Board           |
|  Top-down AI Agent Campus                              |  Queued Planning     |
|                                                         |  Running Verifying   |
|  [Queue] -> [Planning] -> [Execution] -> [Verification] |  Blocked Done        |
|       \                         \                      |----------------------|
|        \                         -> [Blocked Alert]     | Selected Detail      |
|         -> [Rest Area] -> [Archive]                     | Agent / Task / Logs  |
|                                                         |                      |
|  Event Stream Overlay                                  |                      |
+---------------------------------------------------------+----------------------+
```

## 3. 默认园区视角

首屏默认展示一个完整 AI 工作园区，地图不需要用户缩放或拖拽才能理解主流程。

默认可见房间：

- `room_queue`：左下或左侧入口，新任务从这里进入。
- `room_planning`：靠近入口，显示任务板和规划 agent。
- `room_execution`：中央最大房间，显示电脑工位和主要 agent。
- `room_verification`：执行室旁边，显示检查台、清单、验证角色。
- `room_blocked`：右上或右侧高可见区域，使用橙红警报地面和警示灯。
- `room_archive`：右下或底部，显示归档柜和完成动效。
- `rest_area`：边缘休息区，空闲 agent 停留。
- `monitor_wall`：靠近顶部或中央，显示真实会话数量。

默认可见 agent：

- 1 个主调度 agent 在规划室。
- 2 个执行 agent 在执行室。
- 1 个验证 agent 在验证室。
- 1 个阻塞处理 agent 在阻塞区或警报区边缘。
- 1-3 个 mini subagent 在执行室或路径上移动。

默认状态：

- 至少 1 个任务正在执行。
- 至少 1 个任务处于阻塞状态，用来验证警报视觉。
- 至少 1 个任务已完成并显示在归档区或完成列。
- 若未接入真实监控，则顶部状态栏显示“基础监控模式 / 模拟数据运行中”。

## 4. 右侧任务看板

右侧面板上半部分是任务看板，下半部分是选中详情。

MVP 看板固定 6 列：

- `队列中`：新建任务，尚未分配。
- `规划中`：正在拆解需求、生成计划、分配 subagent。
- `执行中`：正在执行代码、命令、文件处理或生成内容。
- `验证中`：正在测试、检查、比对结果。
- `阻塞`：等待用户、权限、依赖、网络、外部输入。
- `完成`：任务完成并进入归档。

任务卡片字段：

- 任务标题。
- 状态徽章。
- 进度条。
- 负责 agent 头像或缩写。
- subagent 数量。
- 最近事件短句。
- 是否来自真实会话或模拟任务。

看板行为：

- 默认按状态分列，不允许第一版自由改列。
- 点击任务会打开详情并高亮场景。
- 阻塞列使用更高视觉权重，但不要遮挡其他列。
- 任务卡片不需要第一版拖拽流转，状态先由模拟数据驱动。

## 5. 详情面板

未选中任何对象时，详情面板显示园区摘要：

- 当前模拟 agent 数。
- 当前真实 Codex / Claude 会话数。
- 运行中任务数。
- 阻塞任务数。
- 最近 5 条事件。

选中任务时，详情面板显示：

- 任务标题、状态、进度。
- 关联 agent 和 subagent。
- 当前所在房间。
- 事件时间线。
- 文件 / 命令摘要占位，不展示真实命令正文。

选中 agent 时，详情面板显示：

- agent 名称、来源、类型、状态。
- 当前任务。
- 所在房间。
- 最近事件。
- 若为真实进程，显示 PID、来源类型、最后发现时间。

选中房间时，详情面板显示：

- 房间名称和用途。
- 当前房间里的 agent。
- 当前房间关联任务。
- 房间对应任务状态。

## 6. 选中态与高亮规则

点击任务：

- 地图中对应 agent 头顶出现任务高亮圈。
- 对应房间边框发光。
- 从任务卡片到地图位置显示一条短暂引导线。
- 详情面板切换为任务详情。

点击 agent：

- agent 放大 1.08 倍或出现柔和描边。
- 关联任务卡片高亮。
- agent 当前路径或目标房间短暂显示。
- 详情面板切换为 agent 详情。

点击房间：

- 房间边界高亮。
- 看板筛选出该状态或该房间关联任务。
- 详情面板切换为房间摘要。

点击空白地图：

- 清除选中态。
- 看板恢复全部任务。
- 详情面板回到园区摘要。

阻塞状态优先级：

- 若选中对象进入阻塞，保持选中态，并在详情面板顶部显示阻塞原因摘要。
- 若多个任务阻塞，顶部状态栏显示阻塞数量，但不弹出模态框打断用户。

## 7. 空状态与基础监控模式

无真实会话时：

- 顶部状态栏显示：`基础监控模式 · 未检测到真实 Claude / Codex 会话`。
- 场景仍运行模拟 agent，不显示错误。
- monitor wall 显示真实会话为 0，模拟 agent 继续工作。

无模拟任务时：

- 队列入口显示空状态图标。
- 看板显示“暂无任务”，但不使用大面积说明文字。
- rest_area 中显示空闲 agent。

进程监控不可用时：

- 顶部状态栏显示：`基础监控模式 · 进程扫描暂不可用`。
- 设置入口显示需要检查权限或运行环境。
- 不阻止用户使用模拟游戏场景。

首次启动时：

- 默认开启模拟数据。
- 默认开启进程级基础扫描。
- 默认不开启终端正文读取。
- 不展示真实命令内容。

## 8. 首屏验收标准

- 打开应用后，不需要任何交互就能看出它是 agent 工作园区。
- 六个核心房间都在默认视角中可见或可通过轻微拖动看到。
- 右侧看板能显示 6 个任务状态。
- 点击任务、agent、房间都能产生明确高亮。
- 无真实进程时，应用仍然可用且不报错。
- 基础监控模式和模拟模式在顶部状态栏明确可见。
- 阻塞状态能在地图、看板和详情面板同时被识别。
