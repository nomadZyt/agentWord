# agentWord First Screen Wireframe

Last updated: 2026-05-29

## 1. 首屏设计目标

首屏要让用户马上感到自己在看一个“本机 AI agent 工作园区”，而不是普通后台监控。

设计优先级：

- 游戏感优先于信息密度。
- 地图场景优先于表格和图表。
- 看板只承担扫描和选中，不抢主场景焦点。
- 阻塞状态必须一眼可见。
- 即使没有真实 Claude / Codex 会话，模拟 agent 也要继续运转。

## 3. 区域尺寸

桌面端：

- 顶部状态栏：56px 高。
- 主地图区域：左侧 68% 宽，剩余高度铺满。
- 右侧面板：固定 400px 宽。
- 任务看板：右侧面板上半部分，约 52% 高。
- 详情面板：右侧面板下半部分，约 48% 高。
- 事件流：地图底部覆盖层，约 140px 高，可折叠。

移动端不是 MVP 范围。若必须适配小屏，先降级为地图全屏 + 底部抽屉。

## 4. 默认地图构图

默认地图从左到右表达任务流：

1. `room_queue` 在左侧入口，显示任务包裹和等待线。
2. `room_planning` 靠近左中，显示任务板、夹板和 PM agent。
3. `room_execution` 位于中间最大区域，放 2-3 个电脑工位。
4. `room_verification` 位于执行室右下或右侧，靠近检查台。
5. `room_blocked` 位于右上，高视觉权重，使用橙红底色和警示线。
6. `room_archive` 位于右下，显示归档柜和完成印章。
7. `rest_area` 放在左下或庭院边缘。
8. `monitor_wall` 放在地图上方或中央边缘，作为真实会话信息看板。
9. `subagent_nest` 靠近执行室，方便 mini subagent 进入执行路径。

## 5. 默认角色摆放

首屏默认角色：

- `white-bow-cat-pm`：规划室，头顶 planning 气泡。
- `tea-cat-engineer`：执行室，头顶 working 气泡。
- `beaver-builder`：执行室，靠近电脑工位。
- `frog-verifier` 或 `japan-style-cat-verifier`：验证室。
- `black-fur-cat-ops` 或 `otter-ops`：阻塞处理室。
- `mini-cat-subagent`：subagent nest 和执行室之间移动。
- 可选：`red-panda-manager` 在规划室旁作为动物团队代表。

默认不要把所有角色都塞进首屏中心，角色之间至少保留 24-32px 的视觉间距。

## 8. 事件流 wireframe

```text
最近事件
[16:28] 新任务进入入口队列
[16:29] white-bow-cat-pm 将任务拆成 3 个子任务
[16:30] mini-cat-subagent 搬运任务卡到执行室
[16:31] beaver-builder 开始执行
[16:32] black-fur-cat-ops 接收到阻塞警报
```

事件流最多显示 5-7 条，避免变成日志墙。

## 9. 高亮规则

点击任务卡：

- 对应任务卡加 2px 高亮边框。
- 对应 agent 头顶出现任务编号气泡。
- 对应房间边框发光。
- 地图镜头轻微平移，让对应 agent 处于主场景中部偏左。

点击 agent：

- agent 放大到 108% 并出现柔和描边。
- 对应任务卡高亮。
- 详情面板切换为 agent 详情。

点击房间：

- 房间边框发光。
- 右侧看板筛选对应状态。
- 详情面板显示房间摘要。

点击地图空白：

- 清除所有选中态。
- 详情面板回到园区摘要。
- 看板恢复全部任务。

## 10. 首屏实现注意

- UI 不要出现大段说明文字。
- 主地图不要被多个浮层遮挡。
- 右侧看板卡片要紧凑，避免像营销页卡片。
- 阻塞警报要醒目，但不要持续闪烁到打扰用户。
- 顶部状态栏只展示关键状态，不放复杂设置。
- 第一版可以先用整张场景背景，不急着切 tilemap。
